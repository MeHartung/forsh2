# Исправление проблемы с определением проекта на Railway

## Проблема
Railway определил проект как Node.js вместо PHP.

## Решение

В настройках проекта на Railway:

1. Откройте проект на Railway
2. Перейдите в Settings → Service
3. В разделе "Build & Deploy" найдите "Builder"
4. Выберите **"Nixpacks"** вместо "Railpack" или "Auto"
5. Сохраните изменения
6. Перезапустите деплой (Redeploy)

## Альтернатива

Если в настройках нет опции выбора билдера:

1. Удалите сервис
2. Создайте новый сервис
3. Выберите "Empty Service"
4. Подключите GitHub репозиторий
5. Railway должен автоматически определить PHP через `nixpacks.toml`

## Проверка

После перезапуска деплоя вы должны увидеть в логах:
- "Detected PHP"
- "Installing composer dependencies"
- "Starting PHP server"
